# helium-browser-avx2
"Hardware-optimized (x86-64-v3/AVX2) build of the ungoogled Helium Browser for Arch/CachyOS.
Helium Browser (AVX2 / x86-64-v3 Optimized)

This is a custom-compiled, hardware-optimized version of the [Helium Browser](https://aur.archlinux.org/packages/helium-browser) (an ungoogled Chromium fork) for Arch Linux and CachyOS. 

Standard AUR binaries are built for generic x86-64 processors. I compiled this release from source specifically using **AVX2 / x86-64-v3** and **-O3** flags to squeeze out maximum Wayland performance and JavaScript rendering speed for modern CPUs.

# Security & Transparency
I built this using Depot.dev cloud servers. The exact `Dockerfile` used for the compilation is included in this repository so you can verify exactly how it was built, the flags that were injected into `makepkg.conf`, and the fix for the AUR package script.

Because this is a pre-compiled binary, you have to trust that the file provided in the Releases tab was actually generated by the included `Dockerfile` without malicious tampering. 

**Why isn't this built using GitHub Actions?**
Chromium/Helium is massive. Compiling it takes over 2.5 hours on a 16-core server. It is too heavy for GitHub's free CI runners. I had to compile this using a 16-core cloud runner via Depot.dev.
