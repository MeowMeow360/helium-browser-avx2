# helium-browser-avx2-pgo
Hardware-optimized (x86-64-v3/AVX2 + PGO) build of the ungoogled Helium Browser for Arch/CachyOS.
This is a custom-compiled, hardware-optimized version of the Helium Browser (an ungoogled Chromium fork) for Arch Linux and CachyOS.
Standard AUR binaries are built for generic x86-64 processors. I compiled this release from source specifically using **AVX2 / x86-64-v3**, **-O3** flags, and **Google's pre-collected Profile-Guided Optimization (PGO)** data to squeeze out maximum Wayland performance and JavaScript rendering speed for modern CPUs.
## Performance Proof (Speedometer 3.1)
Synthetic benchmarks run on an Intel i5-4690T with 16GB RAM running natively on CachyOS (Wayland):
 * Standard CachyOS Chromium baseline: **9.7**
 * This Optimized Helium Build: **11.34** (~16.91% raw rendering uplift)
## Security & Transparency
I built this using Depot.dev cloud servers. The exact Dockerfile used for the compilation is included in this repository so you can verify exactly how it was built, the flags that were injected into makepkg.conf, the PGO GN arguments, and the fixes applied to the AUR package script.
Because this is a pre-compiled binary, you have to trust that the file provided in the Releases tab was actually generated by the included Dockerfile without malicious tampering.
**Why isn't this built using GitHub Actions?** Chromium/Helium is massive. Compiling it takes over 2.5 hours on a 16-core server. It is too heavy for GitHub's free CI runners. I had to compile this using a 16-core cloud runner via Depot.dev.
